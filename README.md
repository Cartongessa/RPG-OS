# 📌RPG-OS: Terminal Hacking Simulator

A text-based RPG simulation game featuring a custom JavaFX graphical terminal interface. The player acts as a cyber-agent infiltrating servers, navigating a dynamically generated file system, exploiting vulnerabilities, and upgrading their neural skills to complete level objectives.

---

## 🚀 How to run the project

### Prerequisites
- Java 25 (LTS)
- Gradle

### Istruzioni

```bash
git clone https://github.com/Cartongessa/RPG-OS.git
cd rpg-os
```

### Build the project
```bash
./gradlew build
```

### Execution
```bash
./gradlew run
```

## 🤖 Use of AI Tools

During the development of this project, the AI was utilized strictly as a support tool to enhance the development process and improve code quality.

Specifically:
* **GitHub Copilot** was used to:
    * Generate repetitive code (e.g., getters, setters, and basic constructors).
    * Auto-complete standard Java syntax to speed up the writing process.


* **Gemini** was used to:
    * Learn and understand the theoretical logic behind specific architectural design patterns (e.g. Model-View-Controller, Composite Pattern).
    * Brainstorm solutions for structural issues, such as identifying and resolving the "Ghost State" bug during the load command execution.
    * Provide suggestions for code refactoring to improve readability and maintainability.
    * Brainstorm ideas for new features or game mechanics to enhance player engagement.

For every architectural advice or code snippet discussed with the AI, I manually adapted the logic to fit my existing project, integrated it, and personally tested it to ensure everything worked flawlessly.
